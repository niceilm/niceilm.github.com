Hadoop의 구성

  • 분산 파일 시스템 (HDFS)
    • 저성능 컴퓨터의 대수를 늘리는 방법
    • 데이터가 분산되어 있어도 빠르게 찾아내고, 고장에 둔감해야 하는 요구 사항을 해결
  • MapRedude
    • 분산 처리 시스템
  • Hadoop ecosystem
    • HDFS와 MapReduce를 사용하기 위한 유틸리티를 총칭
    • Hadoop이 순차적으로 데이터를 사용하는 일괄 처리 방식이기 때문에, 여러 작업을 동시에 할 수 없음 (이해 안 됨 더 조사 필요??)
    • 데이터 유실을 막기 위한 방법 및 파일 시스템 효율성 개선이 필요

Hadoop Module

  • Hadoop Common : 다른 Hadoop모듈을 지원하는 공통 유틸
  • Hadoop Distributed File System (HDFS) : 애플리케이션 데이터 접근에 빠른 속도를 제공하는 분산 파일 시스템
  • Hadoop YARN : 잡 스케쥴링 및 클러스터 리소스 관리 framework (Hadoop 2.0부터 추가)
  • Hadoop MapReduce : 큰 데이터의 병렬 프로세싱을 위한 YARN를 베이스로 한 시스템

Hadoop의 특징

  • 노드가 늘어날 수록 데이터 량/처리 속도가 linear하게 증가
  • 로그, 텍스트, 이미지 등의 비정형 데이터를 다루는 데 유리
  • 실시간 보다는 배치(웹 크롤링, 로그 분석 등)에 적합하다.

2013년 7월 6일 토요일 오후 9시 00분 in Javascript

iTerm2 시작할 때 서버 접속 방법

iterm2 는 osx 용 터미널 프로그램이다. 기본 터미널 프로그램보다 더 다양한 기능을 제공하고 있고 게다가 무료이다.

http://www.iterm2.com/

iterm2의 좋은 기능중 하나인 여러 서버에 동시에 접속하는 방법에 대해서 알아본다.

  1. 우선 접속한 서버별로 Profiles을 추가한다.
  2. Command 를 사용하여 바로 접속을 유도한다.
    접속시에는 SSH 키를 이용해서 암호 없이 접속되도록 하면 편하다.
  3. 터미널을 띄워서 워하는 순서대로 배치를 해둔다.
  4. 상단메뉴에서 Window를 선택하고 현재 Window의 배치를 저장한다.
  5. 다시 설정으로 돌아가 Open default window arragement 를 활성화 시킨다.
  6. 이제부터 iterm2를 실행시키면 저장해둔 배치대로 잘 뜬다.
