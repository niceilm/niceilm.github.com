Hadoop의 구성

  • 분산 파일 시스템 (HDFS)
    • 저성능 컴퓨터의 대수를 늘리는 방법
    • 데이터가 분산되어 있어도 빠르게 찾아내고, 고장에 둔감해야 하는 요구 사항을 해결
  • MapRedude
    • 분산 처리 시스템
  • Hadoop ecosystem
    • HDFS와 MapReduce를 사용하기 위한 유틸리티를 총칭
    • Hadoop이 순차적으로 데이터를 사용하는 일괄 처리 방식이기 때문에, 여러 작업을 동시에 할 수 없음 (이해 안 됨 더 조사 필요??)
    • 데이터 유실을 막기 위한 방법 및 파일 시스템 효율성 개선이 필요

Hadoop Module

  • Hadoop Common : 다른 Hadoop모듈을 지원하는 공통 유틸
  • Hadoop Distributed File System (HDFS) : 애플리케이션 데이터 접근에 빠른 속도를 제공하는 분산 파일 시스템
  • Hadoop YARN : 잡 스케쥴링 및 클러스터 리소스 관리 framework (Hadoop 2.0부터 추가)
  • Hadoop MapReduce : 큰 데이터의 병렬 프로세싱을 위한 YARN를 베이스로 한 시스템

Hadoop의 특징

  • 노드가 늘어날 수록 데이터 량/처리 속도가 linear하게 증가
  • 로그, 텍스트, 이미지 등의 비정형 데이터를 다루는 데 유리
  • 실시간 보다는 배치(웹 크롤링, 로그 분석 등)에 적합하다.

HDFS 구조

  • NameNode (Master)
    • 가용성을 위해 Secondary NameNode가 존재
      • Secondary NameNode는 Primary NameNode의 파일 정보 merge 부하를 줄이고, 문제 발생 시 복구를 위한 백업 용일 뿐 Primary NameNode 문제 발생 시 Secondary가 Primary의 역할을 하는 fail-over의 용도는 아님
      • 링크
    • 모든 파일의 namespace, meta-data, 파일의 chuck 정보를 저장
    • 어느 파일의 chuck가 어느 Datanode에 위치하고 있는지 알고 있음
    • NameNode에서 모든 파일 정보를 관리하므로, 파일 개수가 많아지면 부담이 커짐
  • Datanode (Slave)
    • 대용량의 파일을 chunk라는 단위로 분할하여 Datanode 3개에 분산 저장(복제본 3개가 존재)
    • 보통 chunk의 크기는 64MB

MapReduce

  • LISP의 Map/Reduce 함수에서 차용
  • Restricted parallel programming model
    • 자바 문법으로 된 Map과 Reduce 함수를 작성해야 함
  • Map worker는 저장소에 저장된 데이터를 읽어서 Key, value 형태의 Map 데이터로 변경한다.
  • Reduce worker는 key, value 형태로 들어온 데이터를 취합해서 결과 파일을 만들어 낸다.

MapReduce 구조

  • Job Tracker (Master)
    • 사용자 명령을 받아 데이터를 분석하기 위한 잡을 생성하고, TaskTracker에게 작업을 위임
  • TaskTracker (Slave)
    • 로컬의 파일, DB정보 등을 이용하여 데이터를 분석

Hadoop의 ecosystem(생태계)

Hadoop을 기반으로 한 제품/프로젝트들은 용도에 따라 구분할 수 있다.

  • Apache Zookeeper
    • 분산 환경에서 서버들간 상호 조정이 필요한 서비스를 제공하는 시스템
    • 4가지 기능을 제공
      • 하나의 서버에만 서비스가 집중되지 않도록 분산
      • 하나의 서버에서 처리한 결과를 다른 서버와 동기화해서 데이터 안정성 보장
      • 운영 서버에 문제가 발생해서 서비스를 제공할 수 없는 경우, 다른 대기 중인 서버를 운영서버로 바꿔서 서비스 중지를 방지
      • 분산 환경을 구성하는 서버들의 환경설정을 통합적으로 관리.
  • Apache Oozie
    • Hadoop 작업을 관리하는 워크플로우 및 코디네이터 시스템
    • 자바 웹 어플리케이션 서버이며, MapReduce나 Pig 작업같은 워크플로우를 제어
  • Apache HBase
    • HDFS 기반의 칼럼 데이터 베이스
    • 실시간 랜덤 조회 및 업데이트가 가능
  • Apache Pig
    • 야후에서 개발 후, 아파치 프로젝트로 전환
    • 복잡한 MapReduce 프로그래밍을 대체할 Pig Latin이라는 자체언어를 사용.
    • MapReduce API를 단순화 시키고, SQL과 유사한 형태로 설계
  • Apache Hive
    • 맵리듀스를 SQL과 비슷한 HiveQL을 통해 실행할 수 있도록 도와주는 프로젝트
    • 맵리듀스를 HiveQL을 사용해 수행하고, 결과는 Hadoop에 파일로 저장된다.
  • Apache Mahout
    • Hadoop 기반으로 데이터 마이닝 알고리즘을 구현한 오픈소스
  • Apache HCatalog
    • Hadoop으로 생성한 데이터를 위한 테이블 및 스토리지 관리 서비스
  • Apache  Avro
    • RPC와 데이터 직렬화를 지원
  • Apache  Chunkwa
    • 분산 환경에서 생성되는 데이터를 HDFS에 안정적으로 저장시키는 플랫폼
    • 분산 서버에 설치된 Agent를 실행하고 Collector가 데이터를 받아 HDFS에 저장한다.
    • 데이터 중복 제거는 MapReduce로 처리한다.
    • 야후에서 개발 후, 아파치 인큐베이션에 포함
  • Apache  Flume
    • 서버의 로그를 실시간으로 집계하기 위한 프로젝트
    • Chukwa와 비슷한 기능
    • Chukwa와 차이점은 전체 데이터 흐름을 관리하는 마스터 서버가 존재하여, 데이터 수집, 전달, 저장의 흐름을 동적으로 변경할 수 있다.
    • 클라우드데라에서 개발 후, 아파치 인큐베이션에 포함
  • Facebook Scribe
    • 데이터 수집 플랫폼
    • 데이터를 중앙 집중 서버로 전송해서 HDFS 및 다양한 저장소에 저장한다.
    • HDFS에 저장하기 위해서는 JNI(Java Native Interface)를 사용해야 한다.
  • Apache  Sqoop
    • HDFS, RDBMS, DW, NoSQL 등 다양한 저장소에 대용량 데이터를 신속하게 전송할 수 있다.
    • 오라클 테이블의 정보를 파일로 추출하고, 이를 Hadoop에 관리하거나 그 반대의 일을 수행
    • Oracle, MS-SQL, DB2, MySQL, PostresSQL 등 RDBMS를 지원한다.
  • Hiho (Hadoop In, Hadoop Out)
    • Hadoop data integration
    • Hadoop에서 데이터를 가져오기 위한 SQL를 지정할 수 있고, JDBC 인터페이스를 지원
    • Oracle, MySQL에 데이터 전송할 수 있다.
  • Cloudera Impala
    • Hadoop 기반의 실시간 SQL 질의 시스템
    • MapReduce를 사용하지 않고 자체 개발한 분산 질의 엔진을 사용해 실시간에 가까운 속도의 질의 기능을 제공합니다.
    • 데이터 조회 인터페이스는 HiveQL을 사용한다.
